# TelegramLLM

AI-ассистент для бизнеса в Telegram. Загрузите каталог товаров и документы компании — бот будет отвечать клиентам от вашего имени: находить товары по фото и тексту, отправлять фотографии, консультировать по ценам и наличию. Всё управляется через удобный веб-дашборд.

## Проблема

Малый и средний бизнес тратит часы на однотипные вопросы клиентов в мессенджерах: «Есть ли в наличии?», «Сколько стоит?», «Покажите фото». Нанимать менеджера дорого, а отвечать самому — не масштабируется.

## Решение

TelegramLLM превращает ваш Telegram-аккаунт в умного продавца-консультанта:

- **Загрузите каталог** — товары с фото, ценами, описаниями и остатками
- **Загрузите документы** — прайсы, инструкции, информацию о компании
- **Подключите Telegram** — бот начнёт отвечать клиентам автоматически
- **Контролируйте** — просматривайте переписки, блокируйте нежелательных, управляйте каталогом

Бот общается как живой менеджер: на «Вы», вежливо, по делу. Не выдумывает товары, не называет цены из головы — только реальные данные из вашего каталога.

## Ключевые возможности

### Умный каталог товаров
- Добавляйте товары с несколькими фото, описанием, ценой, размерами, весом и количеством на складе
- Поиск по тексту — клиент пишет «есть тумбы?», бот находит подходящие товары и называет цену
- Поиск по фото — клиент отправляет фотографию, бот распознаёт похожий товар из каталога через CLIP
- Отправка фото товаров — клиент просит «покажи фото», бот отправляет реальные фотографии из каталога
- Подбор альтернатив — «есть подешевле?» → бот ищет товары дешевле из той же категории

### RAG по документам компании
- Загружайте TXT, PDF, DOCX, Markdown файлы
- Документы автоматически разбиваются на чанки и индексируются через sentence-transformers
- Бот использует документы как базу знаний при ответах на вопросы клиентов

### Автоответы в Telegram
- Прямое подключение через Telegram API (MTProto) — не бот-аккаунт, а ваш реальный аккаунт
- Автоматические ответы на входящие личные сообщения
- Поддержка альбомов (несколько фото в одном сообщении)
- Запоминание контекста диалога — бот помнит о чём шёл разговор
- Извлечение фактов о клиенте (имя, город) и персонализация обращения

### Защита от выхода из роли
- Многослойная система предотвращения prompt injection
- Автодетекция ответов в стиле «я — языковая модель» с автоматическим перегенерированием
- Бот никогда не признаётся что он ИИ

### Веб-дашборд
- Управление каталогом товаров (CRUD с загрузкой нескольких фото)
- Загрузка и управление документами базы знаний
- Тестовый чат с ботом прямо из браузера
- Просмотр Telegram-переписок и истории диалогов
- Управление контактами — блокировка, удаление, очистка истории
- Мастер подключения Telegram-аккаунта

## Стек технологий

| Компонент | Технологии |
|---|---|
| **Frontend** | Next.js 16, React 19, Ant Design 6, TypeScript |
| **Backend** | NestJS 11, TypeORM, Passport JWT, TypeScript |
| **AI Service** | FastAPI, SQLAlchemy (async), Python 3.12 |
| **База данных** | PostgreSQL 16 + pgvector (векторный поиск) |
| **LLM** | OpenRouter / OpenAI-совместимые API (Qwen, GPT-4o-mini и др.) |
| **Эмбеддинги текста** | sentence-transformers (all-MiniLM-L6-v2, 384d) — локально |
| **Эмбеддинги изображений** | CLIP ViT-B-32 (512d) — локально через PyTorch |
| **Telegram** | gram.js (MTProto, прямое подключение) |
| **Контейнеризация** | Docker, Docker Compose |

## Как это работает

### Поиск товаров по тексту
```
Клиент: "Есть тумбы?"
    → Эмбеддинг запроса (MiniLM, 384d)
    → Cosine similarity с эмбеддингами товаров в pgvector
    → Top-3 результата с ценами и наличием
    → LLM формирует ответ менеджера
Бот: "Да, в наличии Тумба Босс Стандарт Кашемир серый — 4300 ₽, 4 шт."
```

### Поиск товаров по фото
```
Клиент отправляет фото мебели
    → CLIP-эмбеддинг изображения (512d)
    → Cosine similarity с CLIP-эмбеддингами каталога
    → Найден похожий товар
    → LLM описывает совпадение
Бот: "Похоже на Диван Кинг Велюр Роял — 64 500 ₽"
```

### Отправка фото товаров
```
Клиент: "Можно фото?"
    → Определение что это запрос фото
    → Поиск ранее обсуждаемых товаров
    → LLM: "Отправляю фото Тумбы Босс Стандарт"
    → Автоотправка реальных фото из каталога через Telegram API
```

### RAG по документам
```
Загрузка файла → Извлечение текста (PDF/DOCX/TXT/MD)
    → Чанкинг (512 токенов, 64 overlap)
    → Эмбеддинги (MiniLM, 384d) → pgvector
    → При вопросе: семантический поиск → контекст + LLM → ответ
```

## Быстрый старт

### Требования

- Docker и Docker Compose
- Или: Node.js 20+, Python 3.12, PostgreSQL 16

### 1. Клонировать и настроить

```bash
git clone <repo-url>
cd TelegramLLM
cp .env.example .env
```

Отредактируйте `.env`:

| Переменная | Описание | Обязательно |
|---|---|---|
| `OPENROUTER_API_KEY` | API-ключ OpenRouter (или `OPENAI_API_KEY`) | Да |
| `JWT_SECRET` | Секрет для JWT-токенов (любая длинная строка) | Да |
| `POSTGRES_PASSWORD` | Пароль БД | Да |
| `LLM_PROVIDER` | `openrouter` или `openai` | Нет (default: openrouter) |
| `OPENAI_CHAT_MODEL` | Модель LLM | Нет (default: Qwen3-80B) |

### 2. Запуск (Docker)

```bash
docker compose up -d
```

Поднимет все 4 сервиса. При первом запуске AI Service скачает модели CLIP (~350 MB) и MiniLM (~90 MB).

### 3. Запуск для разработки

```bash
# PostgreSQL + AI Service в Docker
docker compose up -d postgres ai-service

# Backend (отдельный терминал)
cd backend && npm install && npm run start:dev

# Frontend (отдельный терминал)
cd frontend && npm install && npm run dev
```

### 4. Открыть

| Сервис | URL |
|---|---|
| Дашборд | http://localhost:3000 |
| Backend API | http://localhost:4000 |
| AI Service API | http://localhost:8000 |
| Swagger (Backend) | http://localhost:4000/swagger |
| Swagger (AI) | http://localhost:8000/docs |

## Использование

1. **Регистрация** — создайте аккаунт на http://localhost:3000
2. **Загрузите документы** — раздел Files: прайсы, описание компании, FAQ
3. **Добавьте товары** — раздел Products: название, фото, цена, количество, описание
4. **Протестируйте** — раздел Chat: задайте вопросы боту и проверьте ответы
5. **Подключите Telegram** — раздел Telegram: введите API ID и Hash (https://my.telegram.org), подтвердите SMS-код
6. **Включите автоответ** — бот начнёт отвечать клиентам от вашего имени

## API

### Аутентификация
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Авторизация (возвращает JWT) |

### Файлы (база знаний)
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/files/upload` | Загрузка файла (TXT, PDF, DOCX, MD) |
| GET | `/api/files` | Список файлов |
| DELETE | `/api/files/:id` | Удаление файла и его чанков |

### Товары
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/products` | Создание товара (multipart) |
| GET | `/api/products` | Список товаров (поддержка ?search=) |
| GET | `/api/products/:id` | Получение товара |
| PATCH | `/api/products/:id` | Обновление товара |
| DELETE | `/api/products/:id` | Удаление товара |
| GET | `/api/products/:id/image/:imageId` | Изображение товара |

### Чат
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/ai/chat` | Отправка сообщения боту |
| GET | `/api/ai/history` | История чата |
| DELETE | `/api/ai/history` | Очистка истории |

### Telegram
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/api/telegram/credentials` | Сохранение API ID и Hash |
| POST | `/api/telegram/send-code` | Отправка SMS-кода |
| POST | `/api/telegram/verify-code` | Подтверждение кода |
| POST | `/api/telegram/verify-password` | 2FA пароль |
| PATCH | `/api/telegram/auto-reply` | Вкл/выкл автоответ |
| DELETE | `/api/telegram/disconnect` | Отключение |
| GET | `/api/telegram/status` | Статус подключения |
| GET | `/api/telegram/peers` | Список контактов |
| GET | `/api/telegram/peers/:peerId/messages` | Переписка с контактом |
| DELETE | `/api/telegram/peers/:peerId` | Удаление контакта |
| PATCH | `/api/telegram/peers/:peerId/block` | Блокировка контакта |
